Serverless email sending (Vercel + SendGrid)

This project includes a serverless function at `/api/send` that accepts POST requests with JSON `{ name, email, message }` and sends a message using SendGrid.

What I added
- `api/send.js` — Vercel-style serverless function using `@sendgrid/mail`.
- `package.json` — lists `@sendgrid/mail` dependency.
- Updated `contact.html` to post to `/api/send`.
- Updated `js/form.js` to POST JSON and show success/error toasts.

How to deploy (Vercel)
1. Install the Vercel CLI and log in, or use the Vercel web UI.
2. Set environment variables in your Vercel project: 
   - `SENDGRID_API_KEY` — your SendGrid API key
   - `SENDER_EMAIL` — the verified sender email (preferably on your domain, e.g. hello@your-domain.com)
   - `RECIPIENT_EMAIL` — (optional) where messages should be delivered; defaults to `SENDER_EMAIL`.
3. Deploy the project to Vercel (via `vercel` CLI or web UI). Vercel will install dependencies and deploy the serverless function.

DNS / Deliverability steps (important for Gmail Primary placement)
- Verify your sender email or domain in SendGrid.
- Add the SPF and DKIM DNS records SendGrid provides to your domain DNS.
- Wait for verification in SendGrid — this ensures sent messages authenticate properly and improves inbox placement.

Testing locally
- The serverless function requires the `SENDGRID_API_KEY` and `SENDER_EMAIL` env vars. You can test locally by running Vercel dev (`vercel dev`) or by deploying.
